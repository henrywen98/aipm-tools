# fair-copy —— 已并入 humanizer

**这个 skill 停用了（2026-07-17）。** 它做的事——把自己反复改过、混进**过程痕迹**（修订史 / 版本说明 / 行内批注 / 备选写法）和**冷读障碍**（行话 / 求真 / 概念分层 / 术语统一 / 结构）的稿，清成读者第一次冷读就顺的交付稿——已经整体并入 **`humanizer`**。

## 为什么合并

从第一性原理看，fair-copy 和 humanizer 本质是**同一件事**：把文字（尤其 AI 写的）改成**一个真人为特定读者写的、一读就懂的**样子。原来人为切成两个 skill，还得写一堆"别混"的边界。现在统一到一个入口，判准都是**冷读测试**（读者第一次看会不会卡）。

humanizer 现在分三层清：
- **层一 · 机器发声**：去 AI 味 + 去翻译腔（fair-copy 没管的语感层，新并入 CLAUDE.md 的反翻译腔规则）。
- **层二 · 读者读不懂**：换白话 / 说清指代 / 术语统一 / 结构摆顺（= 原 fair-copy 轴二）。
- **层三 · 过程噪音**：修订史 / 批注 / 备选（= 原 fair-copy 轴一）。

## 该用什么

- "清成交付版 / 去掉我的修改记录 / 换成大白话 / 誊清稿" → **`humanizer`**。
- "去 AI 味 / 降 AIGC 率 / 太像 AI 写的 / 翻译腔太重" → **`humanizer`**。
- "读懂别人 / AI 丢来的臃肿稿、边读边改" → **`read-digest`**（input 侧，方向相反）。
