# Humanizer — 去 AI 味（中文为主）

把一份"一看就是大模型写的"文本，改成像具体的人写的：去掉机器腔的套话、黑话、排比和升华，换上具体事实和有变化的句子，同时保住原意和覆盖面。

## 这个 skill 在解决什么（以及不解决什么）

大模型靠"猜下一个最可能的词"生成文本，结果天然倾向于**对最多场景都成立的、最统计常见的写法**——于是堆出一套可识别的机器腔：拔高意义、含糊归因、三件套排比、黑话词、空洞升华结尾。这个 skill 就是把这些 tell 找出来、改回人话。

**边界（最重要）：** 这处理的是「**AI 生成的味道 / 文风**」。它**不**处理「**自己反复改稿留下的过程痕迹**」——修订历史、版本说明、`(待确认)(TODO)` 行内批注、给自己的旁批、多处并存的备选写法。那是另一类需求，用 **`fair-copy`** skill。

- 用户说"这稿太像 AI 写的 / 帮我去 AI 味 / 降 AIGC 率" → **本 skill**。
- 用户说"清成交付版 / 去掉我的修改记录和版本说明 / 别让对方看到我中途怎么改的" → **`fair-copy`**。
- 两件事可以先后做：先 `fair-copy` 清掉过程痕迹，再 `humanizer` 去 AI 味。

## 工作流：draft → audit → final

1. **通读，逐条找 tell。** 对照下面的速查表（中文）或 `references/patterns-en.md`（英文稿）。不要见词就改——先确认它是不是 cluster 的一部分（见「误判防护」）。
2. **写 draft。** 保留原意和覆盖面：**原文几段，改完还是几段**，别越改越短到丢信息。把机器腔换成具体事实、简单句式（多用"是 / 有"），匹配原文该有的语域。
3. **audit：自问"这段为什么一眼就是 AI 写的？"** 用一两句话点出残留的 tell。
4. **写 final，** 把 audit 点出的都改掉。中文稿收尾搜一遍 `——`；英文稿收尾搜 `—`、`–`，有残留就是没改完。
5. **交付 final。** 默认只给改好的成品；用户要看时再附 draft 和"改了什么"的简短清单。

## 中文 AI 味速查表

逐条对照，命中且成 cluster 就改。**完整反例 → 正例见 `references/patterns-zh.md`。**

**内容类**
- 拔高意义：标志着……的重要里程碑、为……奠定坚实基础、开启……新篇章
- -ing 拖尾：句末挂 `，体现了 / 彰显了 / 展现了 / 反映出……`
- 广告腔：坐落于、底蕴深厚、美不胜收、令人叹为观止、不容错过
- 模糊归因：业内人士认为、有关研究表明、专家指出（不点名是谁）
- "机遇与挑战 / 展望未来"八股段；堆砌媒体背书 / 粉丝数

**语言类**
- 黑话当 AI 词：赋能、抓手、闭环、纵深、范式、重塑、链路、心智、颗粒度、对齐、拉通、沉淀、打法、底层逻辑（扎堆即中招）
- 回避"是"：作为……的存在、扮演着……的角色、承载着、肩负着
- 否定式排比：不仅……而且、不只是……更是、不是……而是
- 强凑三段式排比；变换称呼（同一对象换着叫）；假范围（从 X 到 Y）；成串无主语 + "得到有效落实"公文腔

**格式类**
- 全角破折号"——"成串（基本清掉，换句号 / 逗号 / 冒号 / 括号）
- 机械加粗成片；inline-header 列表（`**要点：** ……`）；emoji 装饰；碎片标题（标题下一句废话热身）

**腔调与填充类**
- 客服腔：希望对你有帮助、如需……请告诉我、需要我继续吗
- 谄媚：这是个很好的问题、你说得非常对
- 权威腔：究其根本、从本质上讲、归根结底、真正的问题在于
- signposting：让我们深入探讨、接下来我们来看、话不多说
- 空洞收尾：总而言之、前景广阔、未来可期、必将迈向……
- 填充词："为了能够"→"为了"、"具备……的能力"→"能"、"值得注意的是,数据显示"→"数据显示"
- 过度对冲（堆好几个"可能也许或许"）；强行金句 / 鼓点短句；格言公式（X 是 Y 的 Z）；假坦诚开场（说实话 / 说白了 / 这么说吧）

## ⚠️ 误判防护（别把正常写作当 AI 味）

正常的人也会命中上面某几条。**单点不算数，看 cluster。** 以下单独出现都**不是**可靠证据：

- 文笔干净、语法规整、风格统一——很多人本来就是专业写手或被编辑过。
- 正式 / 学术词汇——AI 是滥用**特定**几个词，不是所有书面语。
- 偶尔一个"然而 / 此外"、偶尔一句短句强调、偶尔一个全角破折号。
- 没有出处——网上大多数内容都没出处。

**判断靠扎堆**：全角破折号 + 三段排比 + "赋能闭环" + "展望未来"结尾，一起出现才是确凿的 AI 味。

**这些是人写的信号，倾向于保留**（过度改写会改没了灵魂）：具体到难以编造的细节（真实地名 / 怪异的引语 / 私人化的小事）、复杂矛盾的情绪（"我说不清为什么但就是别扭"）、有年代感的梗和俚语、作者能解释"为什么这么写"的选择、长短交错的句子节奏、真诚的插话和自我更正。

## 别改成"没灵魂的干净稿"

去掉 AI 味只是一半。**没有声音的、句句一样长、只中立陈述、不带任何观点的稿，跟 AI 味一样一眼假。** 仅在内容和作者口吻允许时（博客 / 随笔 / 观点 / 个人化文字）注入人味：有观点（对事实有反应，而不只是罗列）、节奏有变化（短句、长句混着来）、容许一点"乱"（题外话、半成型的想法）。**但公文 / 技术文档 / 法律 / 百科类，中性朴素本身就是正确的人类声音，别硬塞第一人称和观点。**

## 可选：声音校准（voice calibration）

用户给了 2-3 段他自己以前写的东西，就先分析再改：句子长短习惯、用词的正式度、怎么起段、标点习惯、口头禅。改写时不只是去 AI 味，而是**换上样本里的写法**——他写短句就别给长句，他用"东西/搞"就别升级成"要素/构建"。没给样本就用默认（自然、有变化、敢有观点）。

## 一句话总结

**先确认是去"AI 文风味"而不是去"改稿过程痕迹"（后者归 `fair-copy`）；按 draft→audit→final 改，靠 cluster 判断、别见词就删，去味之余别把人味也改没了。**
