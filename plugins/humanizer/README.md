# humanizer — 去 AI 味（中文为主）

把一份"一眼就是大模型写的"文本，改回像具体的人写的：去掉机器腔的套话、黑话、排比和升华，换上具体事实和有变化的句子，同时保住原意和覆盖面。

## 它解决什么

大模型靠"猜下一个最可能的词"生成文本，天然倾向于**对最多场景都成立的、最统计常见的写法**——于是堆出一套可识别的机器腔。本插件把这些 tell 找出来、改回人话。中文为主，覆盖中文特有的 AI 味；写英文稿时也提供英文专有 tell。

## ⚠️ 和 fair-copy 的区别（别混）

| | humanizer（本插件） | fair-copy |
|---|---|---|
| 处理什么 | **AI 生成的文风味道** | **自己改稿留下的过程痕迹** |
| 典型对象 | 套话 / 黑话 / 排比 / 升华结尾 / 全角破折号 | 修订历史 / 版本说明 / `(待确认)(TODO)` / 旁批 / 备选写法 |
| 触发话术 | "去 AI 味""降 AIGC 率""太像 AI 写的" | "清成交付版""去掉我的修改记录""别让对方看到我中途怎么改的" |

两件事可以先后做：**先 `fair-copy` 清掉过程痕迹，再 `humanizer` 去 AI 味。**

## 触发场景

只要意图是"让文本读起来不像 AI、更像人写的"，就会触发，例如：

- "这段太像 AI 写的，帮我去 AI 味"
- "降一下 AIGC 率 / 降低 AI 率"
- "别这么机器腔，改得自然点"
- "去机器味 / AI 痕迹太重，改得像人写的"

## 怎么用

按 **draft → audit → final** 三步：

1. 通读，对照速查表逐条找 tell（**靠 cluster 判断，别见词就改**）。
2. 写 draft，保留原意和覆盖面（原文几段、改完还是几段）。
3. audit：自问"这段为什么一眼就是 AI 写的？"，点出残留。
4. 写 final 改掉残留，收尾搜一遍全角破折号 `——`。

可选 **声音校准**：给 2-3 段你自己以前写的文字，改写时会换上你的写法（句长、用词、口头禅），而不是泛泛地去味。

## 覆盖的 AI 味（节选）

- **内容类**：拔高意义（"标志着……的重要里程碑"）、模糊归因（"业内人士认为"）、广告腔（"坐落于 / 底蕴深厚"）、"机遇与挑战 / 展望未来"八股、`体现了/彰显了……` 拖尾
- **语言类**：互联网黑话（赋能 / 抓手 / 闭环 / 纵深 / 重塑）、回避"是"、"不仅……而且"、强凑三段排比、变换称呼、"从 X 到 Y"假范围
- **格式类**：全角破折号"——"滥用、机械加粗、inline-header 列表、emoji 装饰、碎片标题
- **腔调与填充类**：客服腔、谄媚、权威腔、signposting、空洞收尾、填充词、过度对冲、强行金句、格言公式、假坦诚开场
- **英文写作专有**：em/en dash 硬规则、curly quotes、Title Case、hyphenated pairs、英文 AI 词表

完整反例 → 正例见 `skills/humanizer/references/patterns-zh.md`（中文）与 `patterns-en.md`（英文）。

## 设计取舍

- **靠 cluster 判断，不靠单点。** 单独一个全角破折号、一个"然而"、一句短句都不算 AI 味；扎堆才算。内置误判防护，避免把正常的人类写作改坏。
- **去味之余保人味。** 句句一样长、毫无观点的"干净稿"和 AI 味一样一眼假。在内容和口吻允许时保留观点与节奏；但公文 / 技术 / 法律 / 百科类，中性朴素本身就是对的。
