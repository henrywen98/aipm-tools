# humanizer — 让 AI 写的文字变成人能读懂的（中文为主）

把 AI 写的、一读就"不像人写的"文字，改成**一个真人为特定读者写的、一读就懂**的稿。去掉机器腔和翻译腔，换白话、说清指代、摆顺结构，清掉自己改稿留下的痕迹——同时保住原意和覆盖面。

> **v2.0（2026-07-17）并入原 fair-copy 插件。** 从第一性原理看，"去 AI 味 / 去翻译腔 / 换白话 / 清成交付版"本质是同一件事：让文字变成人能顺畅读懂的。原来切成 humanizer 和 fair-copy 两个 skill，现在统一成一个入口。

## 它解决什么

大模型靠"猜下一个最可能的词"生成文本，天然堆出一套机器腔：拔高升华、三段排比、黑话、名词化的长句、成串被动、套话开场和空洞收尾——翻成中文更僵。加上作者自己的**知识的诅咒**（看不见只有内行懂的行话、只有自己知道还没做的事、只有自己看得懂的旧版指向），读者一读就卡，作者却看不到。本插件站到"读者第一次冷读"的位置，把这些卡点清掉。

## 判准：冷读测试

逐处只问一句：**目标读者第一次看到这里，会不会停下来？** 会卡就处理（减法默认，能删就删，删不掉换白话），不卡别动。冷读测试**相对具体读者**——同一个词对内行是精确、对外行是黑话，别一刀切往最低水位夯。

## 三层要清的东西

| 层 | 清什么 | 来自 |
|---|---|---|
| **① 机器发声** | 去 AI 味（拔高升华 / 三段排比 / 黑话 / 破折号滥用 / 空洞收尾）+ 去翻译腔（名词化"进行了优化"、长句从句嵌套、滥用被动、一句三个"的"、"值得注意的是 / 总而言之"套话词） | humanizer + Funda 中文写作规矩 |
| **② 读者读不懂** | 行话换白话、抽象词说清指谁、术语前后统一、内部编号自解释、错别字改掉、结构按文档类型摆顺 | 原 fair-copy 轴二 |
| **③ 过程噪音** | 修订史 / 版本说明 / `(待确认)(TODO)` 批注 / 旁批 / 备选写法（只留最终版） | 原 fair-copy 轴一 |

贯穿三层的**求真过滤**：没做的删、做不到的改成现实做法、真·待办保留并诚实标注。

## 触发场景

只要意图是"让文字读起来像人写的、让读者读得懂"，就会触发，例如：

- "这段太像 AI 写的，帮我去 AI 味 / 降 AIGC 率"
- "这份稿翻译腔太重，让中文读起来顺一点"
- "清成交付版 / 去掉我的修改记录和版本说明"
- "把这些行话换成大白话，别人看不懂"

## 怎么用

按 **draft → audit → final** 三步：

1. 先锁定目标稿、定交付对象（读者）。
2. 通读，对照三层逐条找卡点（**AI 味靠 cluster 判断，别见词就改**）。
3. 写 draft，保留原意和覆盖面（原文几段、改完还是几段）。
4. audit：自问"这段为什么还不像人写的 / 读者会卡在哪"，点出残留。
5. 写 final 改掉残留，收尾搜一遍全角破折号 `——`。
6. 两档确认：明确命中项打包直接清；只把求真存疑 / 术语去留 / 场景反转这类真需要判断的单独列出问（≤3 条）。

可选 **声音校准**：给 2-3 段你自己以前写的文字，改写时会换上你的写法（句长、用词、口头禅），而不是泛泛地去味。

## 覆盖的机器味（节选）

- **AI 味 · 内容类**：拔高意义、模糊归因、广告腔、"机遇与挑战 / 展望未来"八股、`体现了/彰显了……` 拖尾
- **AI 味 · 语言类**：互联网黑话（赋能 / 抓手 / 闭环）、回避"是"、"不仅……而且"、强凑三段排比、"从 X 到 Y"假范围
- **AI 味 · 格式类**：全角破折号"——"滥用、机械加粗、inline-header 列表、emoji 装饰、碎片标题
- **AI 味 · 腔调填充类**：客服腔、谄媚、权威腔、signposting、空洞收尾、过度对冲、强行金句、假坦诚开场
- **翻译腔**：名词化 + 万能动词（"进行了 X"）、长句 / 从句套从句、一句三个"的"、空修饰词（非常 / 相当 / 相关的）、机械连接词（以及 / 从而 / 首先其次）
- **英文写作专有**：em/en dash 硬规则、curly quotes、Title Case、hyphenated pairs、英文 AI 词表

完整反例 → 正例见 `skills/humanizer/references/patterns-zh.md`（中文，A–D 为 AI 味、E 为翻译腔）与 `patterns-en.md`（英文）。

## 设计取舍

- **靠 cluster 判断，不靠单点。** 单独一个全角破折号、一个"然而"、一句短句都不算 AI 味；扎堆才算。内置误判防护，避免把正常的人类写作改坏。
- **别削平对真实读者精确的术语。** 给工程的 `fail-closed`、给医生的 `BID`、给律师的"对价"，可能正是最准的词——按真实读者判断，拿不准问。
- **长文档只做定点编辑，别整篇重写**（会引入信息错乱和事实漂移）。
- **去味之余保人味。** 句句一样长、毫无观点的"干净稿"和 AI 味一样一眼假；但公文 / 技术 / 法律 / 百科类，中性朴素本身就是对的。

## 边界

- 区别于 `read-digest`：那是 **input 侧**（读懂别人 / AI 丢来的臃肿稿、边读边改），本插件是 **output 侧**（把自己手上的稿清成人能读懂的成品），方向相反。
- 原 `fair-copy` 已并入本插件（见 `plugins/fair-copy/`，仅保留作重定向）。
